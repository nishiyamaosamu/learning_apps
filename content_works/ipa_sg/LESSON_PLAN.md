# 情報セキュリティマネジメント試験（SG） レッスン計画

全体目標：全8章を **84レッスン** で構築する。
シラバスVer.4.1の重点分野（セキュリティ・法務）を厚く、その他の分野はSG試験で問われる角度（セキュリティ視点）に圧縮する。科目B（技能・ケーススタディ型）は専用章（第8章）を設け、公開問題（令和5〜8年度）の出題型に対応させる。
単語暗記は別機能（暗記カード）に委譲し、各レッスンは **目標／説明＋頻出概念＋計算・図解** を中心に組み立てる。

- 1レッスン ＝ 学習ページ **約8〜11**（クイズ除く）＋ クイズ **約3〜4** ＝ **合計12〜15ページ**。ページ数目安はレッスンの密度（用語数・計算・ケースの重さ）に応じて 12〜15 の幅で設定する。
- 構成の基本：導入 → 説明（テキスト/画像）を一通り → **末尾にミニクイズをまとめて配置**。
- 「ページ」は [docs/LESSON.md](../../docs/LESSON.md) の **scene**（1画面）に対応。`説明`＝narrationシーン、`クイズ`＝quizMultipleChoice / quizFillInTheBlank。
- **ビジュアル**列：そのページの表示モード。`画像`＝静的画像（step の `imageUrl`）／`—`＝テキストのみ。
- **第8章（科目B実践）の基本形**：説明ページでケース（A社の状況）を提示 → 判断ポイントを解説 → クイズで模擬ケース演習。各レッスンは公開問題の出題型（例：R5問13型）に対応付ける。
- **前提知識ページ**：攻撃手法・実装技術のレッスン（L7・L8・L14・L30・L34〜L36）は、前提となる仕組み（HTTP・cookie・DNS・メール配送・IPアドレス/ポート番号など）の軽い説明ページを冒頭に1〜2枚置く。第6章の基礎レッスンより先に学んでも詰まらないようにする。

## 用語の主担当（詳細化時の「定義の正」）

同じ用語が複数レッスンに登場するため、定義・呼称は主担当レッスンに揃える。

- **バックアップ方式（フル/差分/増分・世代管理）と計算**：L51 が正。L19・L28・L37・L73・L74 は L51 の定義を参照する。
- **匿名加工の手法名（項目削除・一般化・トップ（ボトム）コーディング・丸め）**：個人情報保護委員会の呼称に統一し、L32（技術）・L42（法制度）・L79（ケース演習）で一致させる。
- **リスク対応の用語（回避・共有・保有・低減）**：L18 が正。L57（プロジェクトのリスク）・L62（システム化計画のリスク分析）は参照する。
- **「インシデント管理」**：L23（情報セキュリティのインシデントハンドリング）と L58（ITサービスマネジメントのインシデント管理）は**同名の別概念**。両レッスンで相互の違いに必ず触れる。
- **RTO/RPO・スタンバイ構成・稼働率**：稼働率・MTBF/MTTRは L49、RTO/RPO/RLO・スタンバイ構成は L59 が正。L48・L74 は参照する。

- 参考資料：[syllabus_sg_ver4_1.md](syllabus_sg_ver4_1.md)（出題範囲）／[sg_koukai_mondai.md](sg_koukai_mondai.md)（公開問題 R5〜R8）
