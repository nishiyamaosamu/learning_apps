# パターン別の詳細ガイド

すべての記述例は `src/videos/demo/demo.tsx` にある。ここでは間違えやすい点だけ補足する。

## title（タイトルカード）

```ts
{
  pattern: "title",
  series: "ITパスポート講座",          // シリーズ名（講座名）
  title: "企業は何のために\n活動するのか", // \n で2行。1行13文字まで
  keywords: ["経営資源", "企業理念", "CSR"], // 本編で扱う語を3つまで。目立たせない前提の飾り
}
```

- keywords は「本編に出てくる用語の予告」。文にしない、体言止めの単語だけ。

## bullets（箇条書き＋イラスト）

- `marker: "blue"` は覚えてほしい用語に。`"pink"` は動画内で1〜2回だけの最強調。
- `sub` は text の補足説明。text と同じ内容を繰り返さない。
- `illust` は `public/images/<app>/` に実在するファイルのみ（例 `images/ipa_ip/scene-xxx.png`）。`ls public/images/<app>/` で確認してから。
  無ければ省略してよい（箇条書きだけで成立するレイアウトになる）。

## vs（対比）

- **左右の rows は同じ観点（k）を同じ順で並べる**。観点が揃わないと対比にならない。
- 最後の行は「弱点」vs「強み」のように結論の差がつく観点にすると締まる。
- 列タイトルのアイコン（left.icon / right.icon）は省略可。付けるなら両方に付ける。

## flow（ステップフロー）

- `abc` はステップの頭文字や番号（"P" や "1"）。`name` は日本語2〜4文字。
- ハイライトは自動で左から順に移動する。データでの指定は不要。
- 5ステップを超えるなら flow を2シーンに分けるか、layers を検討する。

## matrix（2×2マトリクス）

- `cells` は [左上, 右上, 左下, 右下] の順。
- `tone: "hi"`（primary100）は最重要の1象限、`"lo"`（primary50）は次点。
  tone を付けた象限だけ、説明の流れに合わせて自動で順に点灯する。全部には付けない。
- colLabels が横軸（上）、rowLabels が縦軸（左）。

## layers（階層図）

- 配列は**上の層から**書く（OSIなら 7→1）。積み上げアニメは自動で下から始まる。
- `highlight: true` は「今日の主役」の1層だけ。note と合わせて使うと効果的。
- 階層に順序の意味がないなら bullets を使う。

## graph（計算・グラフ）— 座標系に注意

viewBox は 220×150。**y は下向きが正**（上が小さい値）。軸は自動で描かれる:
- 原点 = (18, 138)、x軸の右端 = 214、y軸の上端 = 8
- 線は x: 18〜208 / y: 18〜138 の範囲に収める

```ts
lines: [
  // role: "guide" = 破線の補助線 / "sub" = 灰色の副系列 / "main" = 青の主系列（最後に描画される）
  { x1: 18, y1: 96, x2: 208, y2: 96, role: "guide", label: "固定費", labelX: 24, labelY: 90 },
  { x1: 18, y1: 96, x2: 205, y2: 56, role: "sub",  label: "総費用", labelX: 172, labelY: 50 },
  { x1: 18, y1: 138, x2: 205, y2: 18, role: "main", label: "売上高", labelX: 170, labelY: 14 },
],
point: { x: 112, y: 78, label: "損益分岐点", labelX: 86, labelY: 66 }, // 交点＝注目点（ピンク）
```

- **point は実際に線が交わる座標に置く**。2直線の交点は手計算で求める
  （線分 (x1,y1)-(x2,y2) をパラメータ表示して連立。ずれると静止画確認で一目で分かる）。
- label の位置は線の右端付近・線と重ならない側に。
- 右側の式カード: `formula` は `\n` 改行で2行まで。`noteHighlight` が結論
  （ピンクマーカーになる）、`noteRest` は補足（「。〜」で始める）。

## term（用語ドン）

- 1動画に1〜2回まで。連発すると効かなくなる。
- `sub` は「正式名称 — 一言定義」の形（例: "Domain Name System — ドメイン名とIPアドレスを対応づけるしくみ"）。

## binary（計算ステップ）

- 2進数以外にも使える（16進数、桁の重みで分解できる計算全般）。
- `weight` の上付き数字は Unicode: ⁰¹²³⁴⁵⁶⁷⁸⁹（例: "2³＝8"）。`<sup>` は使えない。
- digit "0" の桁は自動で減光され、product も灰色になる。データは正直に "0" を書く。

## quiz（動画内クイズ）

- 本編で説明した内容だけから出題する（新情報を出さない）。
- `correct: true` は必ず1つ。正解は A に固定しない（たまに B にする）。
- `foot` は省略すると標準文言。特に書くことがなければ省略でよい。

## 尺の調整

各パターンの尺は内容量から自動計算される（`src/videos/duration.ts`）。
基本は指定不要。ナレーションを想定して特定シーンを長くしたいときだけ
`durationSec` を明示する（アニメーションは先頭から同じタイミングで走る）。
