primary600 primary800 500 300 100 50 correct review incorrect

※ この帯で白文字が載っているのは primary600 / 800 のみ——運用ルール1をこの帯自体が実演している。primary500 と semantic系（correct 3.8:1 / review 2.2:1 / incorrect 3.4:1）は白い小サイズ文字のコントラストが不足するため濃色文字で表記。semantic色上の白は太い記号（○✕マーク等・3:1以上）に限る。

運用ルール

AppColorsのコメントに定める5原則。デザイン判断はすべてこれを優先する。

RULE 1 白い文字は primary600 以上のみ

primary50〜500 の上に白文字を置かない。CTAやバッジで白文字が要る場面は primary600 / primary800 を使う。semantic色（correct / review / incorrect）の上の白は、文字ではなく太い記号（○✕マーク・数字バッジ等、コントラスト3:1以上）に限る。

RULE 2 正誤は 色 + アイコン の二重符号化

正解・不正解を色だけで伝えない。必ず ○ / ✕ アイコンを併記する（色覚多様性への対応）。動画も同じ文法。

RULE 3 ブランド差し替えは primary 系のみ

アプリごとのテーマ変更で触ってよいのは primary 系だけ。semantic（correct / incorrect / review）と accent は全アプリ共通で固定。

RULE 4 アンバーは「学習の促し」専用

review系（アンバー）が指すのは要復習とストリークだけ。課金・プレミアム導線には使わず primary 系で組む（1色に複数の意味を載せない）。ピンク（accent）は装飾・強調のみで、状態表示には使わない。

RULE 5 タップ48dp・フォント拡大に耐える

タップ対象は最小48×48dp（見た目が小さくても当たり判定で確保）。OSのフォント拡大1.3倍まではレイアウトが崩れず、テキストは省略記号ではなく折り返しで逃がす。

影（elevation 2段）

shadow-card shadow-float

基本は border で区切るフラット基調で、影は「持ち上がっているもの」にだけ使う。card = 通常カードの接地感、float = 暗記カード・ダイアログ・スマホモック等の浮遊。ダークでは影が見えないため、面の明度差（surfaceDark > bgDark）で代替する。

レイアウト既定値

48dpタップ対象の最小サイズ（ルール5）。ボタン高・リスト行高の下限 56dpAppBar高、タブバー高（+ホームインジケータのセーフエリア） 16dpコンテンツの左右マージン（画面幅にかかわらず固定） 640dpタブレット等での本文最大幅（中央寄せ、それ以上は伸ばさない）

モーション

duration 3段 × easing 2種だけで組む。動きの役割は「操作の確認（fast）」「結果の提示（base）」「達成の演出（slow）」の3つ。装飾のための常時アニメーションは置かない。デモは実際のトークン値でループ再生している。

選択 — dur-fast 150ms

ウ実績を評価・分析する

--dur-fast + --ease-out。色と枠だけを変え、位置やサイズは動かさない。タップの瞬間に返るのが最優先。

正誤リビール — dur-base 250ms

正解！

--dur-base + --ease-out。下から8dpフェードイン。バナー → 選択肢の色替え → 解説カードの順で、同時ではなく1テンポずつ見せる。

プログレス加算 — dur-slow 400ms

--dur-slow + --ease-in-out。進捗・スコア・「覚えた」率の加算は必ずアニメーションで見せる（達成感はここで作る）。瞬時に置き換えない。

カードフリップ — dur-slow 400ms

SWOT分析
内部・外部×プラス・マイナスの4象限で経営環境を整理する手法

--dur-slow + --ease-in-out の3D回転。OSの「視覚効果を減らす」設定時はフリップをクロスフェードに差し替える。
