# 目的

レッスンは、説明（テキスト・画像）の**コンテンツページ**と、理解度チェックの**ミニクイズ**で学習内容を提供します。
まずコンテンツページを縦スワイプで読み進め、最後に**確認問題（クイズ）に別画面で挑戦**し、**本番の演習（exercises = 過去問）を参照する**構成が一般的です。

- レッスンの一覧情報（カテゴリ・id・title など）は `base.json` 側が持つ。本ドキュメントは**1レッスンの中身**の構造のみを規定する。
- コンテンツは YAML で執筆する（ドキュメント内の例も YAML）。配列の順序がそのまま表示順になる（明示的な番号フィールドは持たない）。
- 音声・画像はアプリの `contents/` 配下に同梱するローカルアセットの相対パスで指定する。

# 表示・操作モデル

スキーマを理解する前提として、レッスンがどう再生されるかを示す。

## ページとクイズの分離

- レッスンは **コンテンツ（`pages`）** と **確認問題（`quizzes`）** を別フィールドに持ち、**表示も2段階に分かれる**：
  1. **コンテンツ feed**: `pages`（説明ページ）を縦スワイプで1ページずつ表示する。
  2. **確認問題画面**: `quizzes`（ミニクイズ）をまとめて出題する**別画面**。コンテンツを読み終えた締めのページの「確認問題に挑戦！」ボタンから遷移する。
- **content ページは複数の [ブロック（block）](#block) を縦に積み上げて一度に表示する**。タップで1つずつ出すような段階表示はしない（ページに入った時点で全ブロックが見えている）。
- ブロックは**テキストと任意の画像**を組み合わせた1単位。画像を持つブロックは**本文の上に画像**を表示する。
- **音声はコンテンツページのみがページ単位で1つ持つ**（`audioUrl`）。クイズは音声を持たない（→ [音声](#音声)）。

```
[コンテンツ feed]（縦スワイプ）
  content 1: block1 + block2 + block3 を一度に表示
  content 2: block4 + block5
  ...
  締めページ: 「確認問題に挑戦！」ボタン → 別画面へ

[確認問題画面]（ボタン操作）
  quiz 1: 設問＋回答UI（回答する → 次へ）
  quiz 2: ...
  完了ページ: 演習（exercises）への参照
```

## 操作（縦スワイプ）

- コンテンツ feed のページ間は**縦スワイプで1ページずつスナップ移動**する。**上スワイプ＝次ページ**／**下スワイプ＝前ページ**。軽いスワイプでも送れるよう判定はやや緩めにする。
- 進む: 次ページへ遷移。最後のコンテンツの次が締めページ（確認問題への導線）。
- 戻る: 前ページへ遷移。戻った先のページは先頭（スクロール位置トップ）から表示する。
- content ページが画面に収まらないときは縦スクロールできる（端まで送ってからのスワイプで次/前ページへ）。
- 確認問題画面ではスワイプは使わず、本文内のボタン（戻る／回答する／次へ）で1問ずつ進める（→ [クイズの進行](#クイズの進行)）。
- AppBar: 左上に閉じる（×）ボタン、残りの領域を**進捗バー**にする（コンテンツ feed・確認問題画面それぞれの進捗）。
- 最下部に細いフッターを置き、中央に音声オン/オフ・倍速などの控えめなメニューを配置する。

## アニメーション

- コンテンツ feed のページ遷移: 縦方向に1ページ分スライドしてスナップする（上＝次、下＝前）。
- 確認問題画面の設問遷移: フェードで切り替える。

## レイアウト

- **content ページ**: ブロックを上から順に縦へ積み上げる。画像を持つブロックは画像（固定高さの上限内）→本文の順。
- 内容が短いページは縦中央寄せ、画面を超えたらスクロール可能にする。

## 音声

- 音声は **content ページのみ**が任意で `audioUrl`（ナレーション）を**1つ**持つ。クイズ（quiz）は音声を持たない（確認問題画面に音声フッターも出さない）。
- 音声は**グローバルのオン/オフトグル**で制御し、状態は設定として永続化する（次レッスンにも引き継ぐ）。倍速も設定として保持する。
- オン（デフォルト）のとき: content ページ入場時に当該ページの `audioUrl` を自動再生。次/前ページへ進んだら前の音声は停止して切り替える（重ねない）。
- オフ→オン切替時: 現在ページの音声を再生開始。
- ページ内に個別の再生ボタンは持たない（聞き直しはトグルの off→on、または前後移動で対応）。

## クイズの進行

- 確認問題画面では、本文内の「回答する」ボタンで採点し、正誤表示後に「次へ」ボタンで次の設問へ進める。
- 未回答のまま「次へ」は出ない＝**進めない**（回答・採点を済ませるまでせき止める）。
- 「戻る」ボタンは未回答・回答後を問わずいつでも可能。
- 戻る→再入場した設問は**回答状態がリセット**される。
- すべての設問を終えると完了ページを表示し、演習（exercises）への参照を示す。

# データ構造

## lesson

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `id` | string | ✅ | レッスン ID |
| `title` | string | ✅ | レッスン名（base.json と重複するが、ファイル単体での可読性のため保持） |
| `pages` | array of [content page](#content-page) | ✅（1つ以上） | コンテンツページの列。縦スワイプで配列順に表示。 |
| `quizzes` | array of [quiz](#quiz) | ー（任意） | 確認問題（ミニクイズ）の列。別画面でまとめて出題する。無ければ空配列。 |
| `exercises` | array of int | ー（任意） | 末尾に表示する本番演習への参照。`exercises/{id}.json` の id を指す。無いレッスンは空配列。 |

- `exercises` は**レッスンの末尾**（確認問題の完了ページ、確認問題が無ければコンテンツの締めページ）に表示される。演習を持たないレッスンも許容する。
- `pages` / `quizzes` には id/title を持たせない（進捗は配列インデックスで表現する）。

## content page

`pages[]` の要素。説明用のページで、1つ以上のブロックを縦に積み上げて一度に表示する（`type` は持たない）。

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `audioUrl` | string | ー（任意） | ページ全体のナレーション音声。ローカルアセット相対パス（例 `lessons/audios/1-1.mp3`）。 |
| `blocks` | array of [block](#block) | ✅（1つ以上） | 縦に積み上げて一度に表示するブロック列。配列順。 |

### block

content ページを構成する1ブロック（テキスト＋任意画像の1単位）。

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `text` | string (Markdown) | ✅ | 本文。Markdown（太字・箇条書き・見出し・インラインコード等）。空文字も許可（画像を見せるだけのブロック等）。 |
| `imageUrl` | string | ー（任意） | このブロックに添える画像。本文の上に表示する。ローカルアセット相対パス（例 `lessons/images/2-1.jpeg`）。 |

## quiz

`quizzes[]` の要素。`type` を discriminator とする**フラット構造**（型ごとのフィールドを同階層に持つ。`data:` でネストしない）。

### type: quizMultipleChoice

複数の選択肢から正解を1つ選ぶミニクイズ。

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `type` | `"quizMultipleChoice"` | ✅ | |
| `question` | string (Markdown) | ✅ | 設問文。 |
| `imageUrl` | string | ー（任意） | 設問に添える画像。 |
| `options` | array of string | ✅（2つ以上） | 選択肢。 |
| `correctOptionIndex` | int | ✅ | 正解の選択肢インデックス（0始まり、範囲内）。 |

### type: quizFillInTheBlank

問題文中の空欄を選択肢で順番に穴埋めするミニクイズ。

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `type` | `"quizFillInTheBlank"` | ✅ | |
| `question` | string | ✅ | 問題文。空欄は `[__]` で表現。 |
| `imageUrl` | string | ー（任意） | 設問に添える画像。 |
| `options` | array of string | ✅ | 選択肢。**同一内容の重複は不可**（各選択肢はユニークな文字列）。 |
| `correctOptionIndices` | array of int | ✅ | 各空欄の正解。`correctOptionIndices[n]` = `question` 中に出現順で数えた n 番目の `[__]` の正解 = `options` のインデックス。 |

ロジック:

- `question` 中の `[__]` を**出現順**に数え、その n 番目の空欄の正解が `correctOptionIndices[n]`。
- 回答 UI は**ドラッグ&ドロップ**。選択肢チップを空欄に配置する。各チップは1つの空欄にのみ使用する。
- したがって `options 数 ≥ 空欄数`。余ったチップが誤答（distractor）になる。`correctOptionIndices` の値は重複しない。

# YAML 例

全ページタイプとブロックの積み上げを網羅した例（後の動作確認サンプルの手本）。

```yaml
id: "2"
title: 企業活動と経営資源
# コンテンツページ（縦スワイプで読み進める）。type は持たない。
pages:
  # 画像付きページ。ブロックを縦に積み上げて一度に表示し、音声はページ単位で1つ。
  - audioUrl: lessons/audios/2-1.mp3
    blocks:
      - text: |
          # 企業活動の目的

          企業は、製品やサービスを社会に提供し、**利益を上げて事業を継続・成長させる**ことを目的に活動します。
        imageUrl: lessons/images/2-1.jpeg
      - text: |
          - 利益だけでなく、社会への貢献も求められる
          - 自社の担当業務を理解するには、まず**企業全体の活動の流れ**を知ることが大切
        imageUrl: lessons/images/2-2.jpeg

  # 画像なしページ（テキストのみのブロックを積み上げ）。音声なし。
  - blocks:
      - text: |
          # 経営資源（ヒト・モノ・カネ・情報）

          企業活動を支える要素を**経営資源**と呼びます。
      - text: |
          - **ヒト**：従業員などの人材
          - **モノ**：設備・原材料・製品
          - **カネ**：資金・お金
          - **情報**：ノウハウ・顧客データ・技術

# 確認問題（別画面でまとめて出題）。type で種別を分ける。
quizzes:
  # 単一選択クイズ。
  - type: quizMultipleChoice
    question: 企業活動の目的として、最も適切なものはどれでしょう？
    options:
      - 利益を上げて事業を継続・成長させること
      - 利益を出さずに活動を続けること
      - 他社より多くの社員を雇うこと
    correctOptionIndex: 0

  # 穴埋めクイズ。
  - type: quizFillInTheBlank
    question: "経営資源は ヒト・[__]・カネ・[__] の4つに整理される。"
    options:
      - モノ
      - 情報
      - 土地
    correctOptionIndices: [0, 1]

exercises: [1]
```
