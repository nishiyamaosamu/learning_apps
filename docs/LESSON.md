# 目的

レッスンは、説明（テキスト・画像）と軽い理解度チェック（ミニクイズ）を**シーン単位**で並べて学習内容を提供します。
最初にテキスト・画像で内容を説明し、途中にミニクイズで軽く確認、**最後に本番の演習（exercises = 過去問）を参照する**構成が一般的です。

- レッスンの一覧情報（カテゴリ・id・title など）は `base.json` 側が持つ。本ドキュメントは**1レッスンの中身**の構造のみを規定する。
- コンテンツは YAML で執筆する（ドキュメント内の例も YAML）。配列の順序がそのまま表示順になる（明示的な番号フィールドは持たない）。
- 音声・画像はアプリの `contents/` 配下に同梱するローカルアセットの相対パスで指定する。

# 表示・操作モデル

スキーマを理解する前提として、レッスンがどう再生されるかを示す。

## シーンとステップ

- レッスンは **シーン（scene）** のフラットな列。シーンは1画面に対応し、**シーンの切り替え時に画面はクリアされてアニメーションで遷移する**（＝シーンは表示のリセット境界）。
- ナレーションシーンは複数の **ステップ（step）** を持つ。ステップは1タップで1つずつ出現し、**同一シーン内では下に累積**していく（前のステップは消えない）。
- シーンが変わると累積はクリアされる。画像の有無に関係なく、テキストのみ → テキストのみのシーン切り替えもある。
- **シーンのモード（テキストのみ / 画像あり）はシーン静的に決まる**：いずれかの step が `imageUrl` を持てば「画像モード」、どの step も持たなければ「テキストのみ」。表示状態（どこまでタップしたか）には依存しない。
- 画像は**ステップ単位で指定・差し替え**する（→ [step](#step) の `imageUrl`）。表示中の画像は「表示済みステップのうち直近で `imageUrl` を指定したステップの画像」で、無指定のステップは直前の画像を継承する。画像モードのシーンは**先頭 step から画像を持つ**（途中から画像が出る混在は不可。データ生成バリデーションで担保する）。
- ステップで画像が切り替わるときは、旧画像を下に敷いたまま新画像を**左→右へワイプ（リビール）**して重ね、描き足された部分が左から順に現れるように見せる。テキスト累積はクリアされない。

```
シーン1: step1                 （タップ）
シーン1: step1 + step2         （タップ＝シーン末尾なので次シーンへ）
シーン2: step3                 （シーン切替で画面クリア＋アニメ）
シーン2: step3 + step4
```

## 操作（タップゾーン）

- 本文エリアを左右に分割。**右70%タップ＝進む**／**左30%タップ＝戻る**（リーダー的操作）。
- 進む: シーン内に未表示ステップがあれば次ステップを出す。全ステップ表示済みなら次シーンへ遷移。最後のシーンの最終ステップでさらに進む＝完了ページ。
- 戻る: **シーン単位**で前シーンへ。戻った先のシーンは**先頭ステップだけの状態にリセット**され、また右タップで1つずつ展開する（ステップ単位の巻き戻しはしない）。最初のシーンでの左タップは無反応。
- AppBar: 左上に閉じる（×）ボタン、残りの領域を**シーン単位の進捗バー**にする。
- 最下部に細いフッターを置き、中央に控えめな**音声オン/オフトグル**のみを配置する（左右タップゾーンはその上の本文エリア）。

## アニメーション

- ステップ出現（同一シーン内）: 下からフェードイン＋スライドアップ。
- シーン遷移: わずかに横へ動かしつつフェードイン/アウト（進む＝右から、戻る＝左から）。移動量は控えめでフェード主体。
- シーン内のステップ画像の差し替えは、シーン遷移とは別に左→右ワイプで行う。

## レイアウト

- **画像モードのシーン**: 画像を**上部に固定高さ枠でピン留め**（その枠はシーン開始時から確保し、画像の縦横比差や差し替えでレイアウトが動かないようにする）。その下にステップが上から累積する。画像差し替えは枠内で左→右ワイプ。
- **テキストのみのシーン**: 画像枠を持たず、ステップ群を**縦中央寄せ**で表示する。
- ステップの領域は**全ステップ分を最初から確保**し（未表示は透明）、新ステップ出現で表示済みテキストの位置がずれないようにする。
- テキストが画面を超えたらテキスト領域をスクロール可能にし、新ステップ出現時は**最新ステップが見える位置へ自動スクロール**する。

## 音声

- 各ステップ・各クイズは任意で `audioUrl`（ナレーション/設問読み上げ）を持つ。
- 音声は**グローバルのオン/オフトグル**で制御し、状態は設定として永続化する（次レッスンにも引き継ぐ）。
- オン（デフォルト）のとき: ステップ出現時・シーン入場時に当該音声を自動再生。途中で次へ進んだら前の音声は停止して切り替える（重ねない）。クイズシーンは入場時に `audioUrl` があれば自動再生。
- オフ→オン切替時: 現在ステップの音声を再生開始。
- 個別ステップの再生ボタンは持たない（聞き直しはトグルの off→on、または戻って再展開で対応）。

## クイズの進行

- クイズシーンはタップ送りの対象外。本文内の「回答する」ボタンで採点し、正誤表示後に右タップで次シーンへ進める。
- 未回答のまま右タップしても**進めない**（回答・採点を済ませるまでせき止める）。
- 左タップ（戻る）は未回答・回答後を問わずいつでも可能。
- 戻る→再入場したクイズは**回答状態がリセット**される（シーンの先頭リセット方針に合わせる）。

# データ構造

## lesson

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `id` | string | ✅ | レッスン ID |
| `title` | string | ✅ | レッスン名（base.json と重複するが、ファイル単体での可読性のため保持） |
| `scenes` | array of [scene](#scene) | ✅（1つ以上） | シーンの列。配列順に表示。 |
| `exercises` | array of int | ー（任意） | 末尾に表示する本番演習への参照。`exercises/{id}.json` の id を指す。無いレッスンは空配列。 |

- `exercises` は**常にレッスンの末尾**（完了ページ）に表示される（シーン列の中には入れない）。導入だけのレッスンなど、演習を持たないレッスンも許容する。
- シーンには id/title を持たせない（進捗は配列インデックスで表現する）。

## scene

各シーンは `type` を discriminator とする**フラット構造**（型ごとのフィールドを同階層に持つ。`data:` でネストしない）。

### type: narration

説明用のシーン。1つ以上のステップを持つ。画像は持たず、各ステップの `imageUrl` で指定する（→ [step](#step)）。

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `type` | `"narration"` | ✅ | |
| `steps` | array of [step](#step) | ✅（1つ以上） | タップごとに1つずつ出現し、シーン内で累積する。配列順。 |

- シーンが「画像モード」か「テキストのみ」かは、いずれかの step が `imageUrl` を持つかで決まる（→ [表示・操作モデル](#表示操作モデル)）。
- **バリデーション（データ生成時に担保）**: 画像モードのシーンは**先頭 step が `imageUrl` を持つ**こと（途中から画像が現れる混在を禁止）。テキストのみシーンは**どの step も `imageUrl` を持たない**こと。

### step

ナレーションシーンの1ステップ（1タップで出現する1単位）。

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `text` | string (Markdown) | ✅ | 本文。Markdown（太字・箇条書き・見出し・インラインコード等）。空文字も許可（画像を見せるだけのステップ等）。 |
| `audioUrl` | string | ー（任意） | 本文ナレーション音声。ローカルアセット相対パス（例 `lessons/audios/2-1.mp3`）。 |
| `imageUrl` | string | ー（任意） | このステップ以降に表示する画像。無指定なら直前の画像を継承する。どの step も持たなければテキストのみシーン。画像モードのシーンは先頭 step が指定する。切替は左→右のワイプ。ローカルアセット相対パス（例 `lessons/images/2-1.jpeg`）。 |

### type: quizMultipleChoice

複数の選択肢から正解を1つ選ぶミニクイズ。

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `type` | `"quizMultipleChoice"` | ✅ | |
| `question` | string (Markdown) | ✅ | 設問文。 |
| `imageUrl` | string | ー（任意） | 設問に添える画像。 |
| `audioUrl` | string | ー（任意） | 設問読み上げ音声。 |
| `options` | array of string | ✅（2つ以上） | 選択肢。 |
| `correctOptionIndex` | int | ✅ | 正解の選択肢インデックス（0始まり、範囲内）。 |

### type: quizFillInTheBlank

問題文中の空欄を選択肢で順番に穴埋めするミニクイズ。

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| `type` | `"quizFillInTheBlank"` | ✅ | |
| `question` | string | ✅ | 問題文。空欄は `[__]` で表現。 |
| `imageUrl` | string | ー（任意） | 設問に添える画像。 |
| `audioUrl` | string | ー（任意） | 設問読み上げ音声。 |
| `options` | array of string | ✅ | 選択肢。**同一内容の重複は不可**（各選択肢はユニークな文字列）。 |
| `correctOptionIndices` | array of int | ✅ | 各空欄の正解。`correctOptionIndices[n]` = `question` 中に出現順で数えた n 番目の `[__]` の正解 = `options` のインデックス。 |

ロジック:

- `question` 中の `[__]` を**出現順**に数え、その n 番目の空欄の正解が `correctOptionIndices[n]`。
- 回答 UI は**ドラッグ&ドロップ**。選択肢チップを空欄に配置する。各チップは1つの空欄にのみ使用する。
- したがって `options 数 ≥ 空欄数`。余ったチップが誤答（distractor）になる。`correctOptionIndices` の値は重複しない。

# YAML 例

全シーンタイプと複数ステップを網羅した例（後の動作確認サンプルの手本）。

```yaml
id: "2"
title: 企業活動と経営資源
scenes:
  # 画像付きナレーション（先頭 step が画像を持つ＝画像モード）。複数ステップで累積表示。
  - type: narration
    steps:
      - text: |
          # 企業活動の目的

          企業は、製品やサービスを社会に提供し、**利益を上げて事業を継続・成長させる**ことを目的に活動します。
        imageUrl: lessons/images/2-1.jpeg
        audioUrl: lessons/audios/2-1.mp3
      - text: |
          - 利益だけでなく、社会への貢献も求められる
          - 自社の担当業務を理解するには、まず**企業全体の活動の流れ**を知ることが大切
        audioUrl: lessons/audios/2-2.mp3
        # このステップで画像を差し替え（左→右ワイプ）。テキストは累積したまま。
        imageUrl: lessons/images/2-2.jpeg

  # テキストのみのナレーション（画像なし・複数ステップ）。
  - type: narration
    steps:
      - text: |
          # 経営資源（ヒト・モノ・カネ・情報）

          企業活動を支える要素を**経営資源**と呼びます。
      - text: |
          - **ヒト**：従業員などの人材
          - **モノ**：設備・原材料・製品
          - **カネ**：資金・お金
          - **情報**：ノウハウ・顧客データ・技術

  # 単一選択クイズ（設問文を自身で持つ）。
  - type: quizMultipleChoice
    question: 企業活動の目的として、最も適切なものはどれでしょう？
    options:
      - 利益を上げて事業を継続・成長させること
      - 利益を出さずに活動を続けること
      - 他社より多くの社員を雇うこと
    correctOptionIndex: 0

  # 穴埋めクイズ。
  - type: quizFillInTheBlank
    question: "経営資源は ヒト・[__]・カネ・[__] の4つに整理される。"
    options:
      - モノ
      - 情報
      - 土地
    correctOptionIndices: [0, 1]

exercises: [1]
```
